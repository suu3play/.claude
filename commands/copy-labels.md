# ラベルコピーコマンド

## 概要

`.claude/labels.md`で定義された標準ラベル設定を指定されたプロジェクトまたは全プロジェクトにコピーするスラッシュコマンド

## 実行手順

1. **標準ラベル定義の読み込み**

    - `.claude/labels.md`ファイルを読み込み
    - カスタムラベル（Type系、Priority系、緊急度系）を抽出
    - ラベル名、説明、カラーコードを解析

2. **対象プロジェクトの確認**

    - 個別指定: 指定プロジェクトが Git リポジトリであることを確認
    - all 指定: 作業ディレクトリ配下を自動スキャンしてプロジェクト一覧を取得
    - gh CLI でアクセス可能であることを確認

3. **標準ラベル作成**

    - labels.mdから読み取ったカスタムラベルを適用
    - Type 系：ドキュメント更新、バグ、リファクタリング、新機能・既存改修、good first issue、作業、質問・相談
    - Priority 系：最優先、中、低
    - 緊急度系：高、中、低
    - 色設定とラベル説明も統一適用

4. **適用確認**
    - ラベル一覧の確認
    - 適用結果の報告

## 使用方法

### 個別プロジェクトへの適用

```
/copy-labels [対象プロジェクト名]
```

### 全プロジェクトへの一括適用

```
/copy-labels all
```

## 実行例

### 個別適用

```
/copy-labels source-flow
/copy-labels portfolio
/copy-labels hobby-weather
/copy-labels finder-scope
```

### 一括適用

```
/copy-labels all
```

上記コマンドは作業ディレクトリ配下の全プロジェクトに標準ラベル体系を適用します。
対象プロジェクトは実行時に自動検出されます。

## 標準ラベル体系

標準ラベルの詳細定義は `.claude/labels.md` で管理されています。
このファイルを編集することで、全プロジェクトに適用されるラベル体系を一元管理できます。

### 現在定義されているラベル

- **Type系ラベル（7種類）**: 作業の種類を分類
- **Priority系ラベル（3種類）**: 優先度を示す
- **緊急度系ラベル（3種類）**: 質問・相談の回答期限を示す

詳細は [labels.md](../labels.md) を参照してください。

## ラベル定義の管理方法

### ラベルの追加・変更

1. `.claude/labels.md`を編集
2. マークダウンテーブルにラベル情報を追加・変更
   - ラベル名
   - 説明
   - カラーコード（#で始まる6桁の16進数）
3. `/copy-labels all`で全プロジェクトに反映

### labels.mdのフォーマット

```markdown
| ラベル名 | 説明 | カラーコード |
|---------|------|------------|
| Type：新しいラベル | 説明文 | #RRGGBB |
```

## 注意事項

-   `.claude/labels.md`が標準ラベル定義ファイル
-   GitHubの書き込み権限が必要
-   既に存在するラベルはスキップされます（エラーは無視）

### all 引数使用時の特別な注意

-   **実行時間**: 複数プロジェクトへの適用のため時間がかかります
-   **エラー処理**: 一部プロジェクトで失敗しても他のプロジェクトは継続処理
-   **自動検出**: 作業ディレクトリ配下のGitリポジトリを動的にスキャン
-   **除外対象**: .claudeプロジェクト自体は除外されます
