# /create-beginner-issues - 初級者向け Issue 作成コマンド

## 概要

初級者が取り組みやすい小規模で明確な Issue を一括作成するカスタムスラッシュコマンド

## 機能

-   初級者向けの単純で明確なタスクの Issue 作成
-   `good first issue`ラベルの自動付与
-   詳細で具体的な実装指示を含む
-   複雑な修正を避けた単一機能の改修
-   最大 3 つ以内の小分類に収まる範囲

## 使用方法

```
/create-beginner-issues [プロジェクト名] [作成個数]
```

### 引数

-   **プロジェクト名** (必須): 対象プロジェクトのディレクトリ名
-   **作成個数** (オプション): 作成する Issue の個数 (デフォルト: 5 個)

### 例

```bash
/create-beginner-issues portfolio
/create-beginner-issues portfolio 3
/create-beginner-issues my-app 7
```

## コマンド実装ロジック

以下の手順で初級者向け Issue を作成する：

1. **引数解析**: プロジェクト名と作成個数を取得（個数未指定時は 5 個）
2. **プロジェクト確認**: 指定されたプロジェクトディレクトリの存在確認
3. **Issue テンプレート選択**: 下記テンプレートからランダムに指定個数を選択
4. **GitHub Issue 作成**: `gh issue create`コマンドで Issue 作成
5. **ラベル付与**: `good first issue`ラベルを自動付与
6. **結果報告**: 作成された Issue 一覧を表示

## 作成される Issue の種類

### UI/UX 改善系

1. **アニメーション効果の追加**

    - ボタンホバーエフェクト
    - ページ遷移時のフェードイン
    - スクロール時の要素表示

2. **レスポンシブ対応の微調整**

    - 特定画面サイズでの余白調整
    - モバイル表示の改善
    - タブレット表示の最適化

3. **アクセシビリティ向上**
    - alt 属性の追加・改善
    - キーボードナビゲーション対応
    - カラーコントラスト改善

### コンテンツ系

4. **テキスト・表示内容の改善**

    - エラーメッセージの分かりやすい文言変更
    - プレースホルダーテキストの改善
    - 説明文の追加

5. **画像・アイコンの追加**
    - ファビコンの設定
    - ローディングアイコンの改善
    - デフォルト画像の追加

### 機能改善系

6. **バリデーション機能の追加**

    - フォーム入力チェック
    - エラー表示の改善
    - 成功メッセージの追加

7. **設定・環境改善**

    - 環境変数の整理
    - 設定ファイルの改善
    - ドキュメントの充実

8. **パフォーマンス向上**
    - 画像圧縮
    - 不要コードの削除
    - 読み込み速度改善

## Issue 作成時の構成

### タイトル形式

```
{カテゴリ}: {具体的な内容}
```

### 本文構成

```markdown
## 概要

{内容}改善タスク

## 詳細

{具体的な実装内容}

## 実装手順

1. {ステップ 1}
2. {ステップ 2}
3. {ステップ 3}

## 受け入れ条件

-   [ ] {条件 1}
-   [ ] {条件 2}
-   [ ] {条件 3}

## 参考資料

<!-- 関連ドキュメントや実装例のURLを後程追加してください -->

## 注意点

-   {初級者向けの注意事項}
-   {よくある間違いの回避方法}

**初回コントリビューター歓迎！質問があればお気軽にコメントしてください。**
```

## ラベル設定

-   `good first issue` (必須)
-   `Type：新機能・既存改修` (機能改善の場合)
-   `Type：ドキュメント更新` (ドキュメント系の場合)
-   `Type：UI・UX改善` (UI 改善の場合)

## 制約事項

-   複雑なロジック変更は含まない
-   データベース設計変更は対象外
-   外部 API 連携は対象外
-   認証・セキュリティ関連は対象外
-   大規模なリファクタリングは対象外

## Issue テンプレートデータ

### 1. ボタンホバーエフェクトの改善

```
カテゴリ: UI/UX改善
タイトル: ボタンホバーエフェクトの改善
詳細: 現在のボタンにマウスホバー時のアニメーション効果を追加

実装手順:
1. 対象ボタンコンポーネントを特定
2. CSSまたはTailwindCSSでhover効果を追加
3. transition-durationを0.3秒に設定
4. 色変更、スケール変更、影効果のいずれかを実装

受け入れ条件:
- ボタンホバー時にスムーズなアニメーションが表示される
- アニメーション時間が適切（0.2-0.4秒）
- すべてのボタンで一貫したエフェクト

ラベル: good first issue, Type：UI・UX改善, Type：新機能・既存改修
```

### 2. 画像の alt 属性改善

```
カテゴリ: アクセシビリティ
タイトル: 画像のalt属性改善
詳細: 不足している画像のalt属性を追加・改善

実装手順:
1. 全ての<img>タグを確認
2. alt属性が空白または不適切な画像を特定
3. 画像の内容を適切に説明するalt属性を追加
4. 装飾的な画像にはalt=""を設定

受け入れ条件:
- 全ての意味のある画像に適切なalt属性が設定されている
- 装飾的な画像にはalt=""が設定されている
- alt属性の内容が画像を適切に説明している

ラベル: good first issue, Type：新機能・既存改修
```

### 3. エラーメッセージの分かりやすい表示

```
カテゴリ: コンテンツ改善
タイトル: エラーメッセージの分かりやすい表示
詳細: フォームのエラーメッセージをより分かりやすい文言に変更

実装手順:
1. 現在のエラーメッセージを確認
2. 技術的すぎる表現を一般的な表現に変更
3. 解決方法のヒントを含むメッセージに修正
4. エラーの種類別に適切なメッセージを設定

受け入れ条件:
- エラーメッセージが一般ユーザーにも分かりやすい
- 解決方法のヒントが含まれている
- 丁寧で親しみやすい文調になっている

ラベル: good first issue, Type：UI・UX改善, Type：新機能・既存改修
```

### 4. ファビコンの設定

```
カテゴリ: パフォーマンス
タイトル: ファビコンの設定
詳細: ブラウザタブに表示されるファビコンを設定

実装手順:
1. 32x32px、16x16pxのアイコンを作成
2. public/フォルダにfavicon.icoとして保存
3. index.htmlのheadタグに<link>要素を追加
4. ブラウザで表示確認

受け入れ条件:
- ファビコンがブラウザタブに表示される
- 複数サイズ（16px、32px）のアイコンが用意されている
- サイトのテーマに合ったデザイン

ラベル: good first issue, Type：新機能・既存改修
```

### 5. モバイル表示での余白調整

```
カテゴリ: レスポンシブ
タイトル: モバイル表示での余白調整
詳細: スマートフォンでの表示時の余白を適切に調整

実装手順:
1. モバイル表示（375px幅）で確認
2. 左右の余白が狭すぎる箇所を特定
3. px-4やpx-6クラスで適切な余白を設定
4. 複数デバイスサイズで表示確認

受け入れ条件:
- モバイル表示で適切な余白が確保されている
- テキストが画面端に寄りすぎていない
- 読みやすいレイアウトになっている

ラベル: good first issue, Type：UI・UX改善, Type：新機能・既存改修
```

### 6. ローディング表示の追加

```
カテゴリ: コンテンツ改善
タイトル: ローディング表示の追加
詳細: データ読み込み中のローディング表示を追加

実装手順:
1. 読み込み処理が発生する箇所を特定
2. スピナーまたはプログレスバーのコンポーネントを作成
3. 読み込み状態の管理（useState使用）
4. 読み込み中の表示切り替えを実装

受け入れ条件:
- 読み込み中にローディング表示が表示される
- 読み込み完了後に適切に非表示になる
- ローディング表示が見やすくデザインされている

ラベル: good first issue, Type：UI・UX改善, Type：新機能・既存改修
```

### 7. README の改善

```
カテゴリ: ドキュメント
タイトル: READMEの改善
詳細: プロジェクトのREADMEファイルに不足情報を追加

実装手順:
1. 現在のREADME.mdを確認
2. 不足している情報（インストール手順、使用方法など）を特定
3. セクションを整理して分かりやすく構成
4. 必要に応じてスクリーンショットを追加

受け入れ条件:
- インストール手順が明確に記載されている
- 使用方法の説明が分かりやすい
- プロジェクトの概要が適切に説明されている

ラベル: good first issue, Type：ドキュメント更新
```

### 8. フォーカス状態の視覚的改善

```
カテゴリ: UI/UX改善
タイトル: フォーカス状態の視覚的改善
詳細: キーボードナビゲーション時のフォーカス表示を改善

実装手順:
1. フォーカス可能な要素（ボタン、リンク、入力欄）を確認
2. focus:リング表示やアウトライン表示を追加
3. フォーカス時の色やスタイルを統一
4. タブキーでの移動順序を確認・調整

受け入れ条件:
- フォーカス時に明確な視覚的表示がある
- 全てのインタラクティブ要素でフォーカス表示が統一されている
- タブキーでの移動が論理的な順序になっている

ラベル: good first issue, Type：UI・UX改善
```
