# 標準ラベル設定

## 概要
claude-configプロジェクトの標準ラベル体系。全プロジェクトで一貫したラベル管理を実現する。

## Type系ラベル

### Type：ドキュメント更新 (#0075ca)
ドキュメントの作成・修正・翻訳に関する作業。READMEや設計書の更新も含みます。

### Type：バグ (#d73a4a)  
不具合・バグ報告。想定外の挙動やエラーが対象です。

### Type：リファクタリング (#cfd3d7)
機能に影響しないコードの改善や整理。可読性や保守性を向上させます。

### Type：新機能・既存改修 (#0e8a16)
新機能の追加、または既存機能の改善。ユーザー価値を向上させる開発です。

### Type：good first issue (#7057ff)
初級の開発者でも取り組みやすいとされる問題。 具体的には、小さなバグの修正や機能には影響しないテキスト変更などが該当。

### Type：作業 (#27cdee)
その他の一般的な作業項目。明確に分類できないタスクや調査などに使用します。

### Type：質問・相談 (#128A0C)
技術的な質問や開発に関する相談。疑問点や実装方法などについて議論したい内容です。

## Priority系ラベル  

### Priority：最優先 (#b60205)
即対応が必要な最優先事項。重大なバグや、納期直前の重要項目です。

### Priority：中 (#f66a0a)
優先度は高いが緊急ではないタスク。リリースやマイルストーンに影響する可能性があります。

### Priority：低 (#fbca04)
対応は必要だが、後回しにできる低優先のタスク。改善系や将来的な対応も含みます。

## 緊急度系ラベル（質問・相談用）

### 緊急度：高 (#FF6B6B)
至急または高優先度で回答が必要な質問・相談。1-2日以内の対応を希望。

### 緊急度：中 (#FFD93D)
中程度の優先度。1週間以内の回答を希望する質問・相談。

### 緊急度：低 (#4ECDC4)
時間に余裕がある質問・相談。回答タイミングは柔軟で問題なし。

## 使用方法

### 基本的な組み合わせ
- `Type：バグ` + `Priority：最優先`
- `Type：新機能・既存改修` + `Priority：中`
- `Type：リファクタリング` + `Priority：低`
- `Type：質問・相談` + `緊急度：高/中/低`

### ラベル適用の指針
1. **必須**: 全issueにType系ラベルを1つ適用
2. **推奨**: 重要度に応じてPriority系ラベルを適用
3. **継続性**: プロジェクト間で一貫した適用基準を維持

## プロジェクトへの適用

### /copy-labelsコマンド
```bash
/copy-labels [プロジェクト名]
```

### 手動適用
各ラベルをgh CLIまたはGitHub UIで個別作成する場合の設定値は上記の通り。

## メンテナンス方針

- **claude-configプロジェクト**: 標準ラベル体系の管理とメンテナンス
- **適用プロジェクト**: 標準体系に従ったラベル運用
- **更新時**: claude-configでの変更後、各プロジェクトに一括反映
