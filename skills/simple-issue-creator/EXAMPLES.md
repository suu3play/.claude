# Simple Issue Creator - 使用例

このスキルの具体的な使用例を示します。

## 例1: シンプルなバグ報告

### ユーザーの要望
「ログインできないバグをIssueにしたい」

### 対話フロー

```
ユーザー: ログインできないバグをIssueにしたい

アシスタント: Issueのタイトルを教えてください。

ユーザー: ログイン機能が動作しない

アシスタント: Issue種別を選択してください:
- バグ
- 新機能
- ドキュメント
- リファクタリング
- 質問・相談
- その他

ユーザー: バグ

アシスタント: 優先度を選択してください:
- 最優先
- 高
- 中
- 低

ユーザー: 高

アシスタント: 問題の詳細を教えてください。

ユーザー: メールアドレスとパスワードを入力してログインボタンを押すと、500エラーが表示されてログインできない。

アシスタント: 再現手順があれば教えてください（なければ「スキップ」と入力）。

ユーザー: スキップ

アシスタント: 期待する動作があれば教えてください（なければ「スキップ」と入力）。

ユーザー: 正常にログインしてダッシュボードに遷移する

アシスタント: その他追加情報があれば教えてください（なければ「スキップ」と入力）。

ユーザー: スキップ

アシスタント: 以下の内容でIssueを作成します:

---
# タイトル
ログイン機能が動作しない

## 問題の詳細
メールアドレスとパスワードを入力してログインボタンを押すと、500エラーが表示されてログインできない。

## 期待する動作
正常にログインしてダッシュボードに遷移する

ラベル: Type：バグ, Priority：中
---

この内容でIssueを作成しますか？ (yes/no)

ユーザー: yes

アシスタント: Issue #52 を作成しました: https://github.com/user/repo/issues/52
```

## 例2: 機能追加リクエスト（詳細あり）

### ユーザーの要望
「CSVエクスポート機能が欲しい」

### 対話フロー

```
ユーザー: CSVエクスポート機能が欲しい

アシスタント: Issueのタイトルを教えてください。

ユーザー: ユーザーリストのCSVエクスポート機能を追加

アシスタント: Issue種別を選択してください:
- バグ
- 新機能
- ドキュメント
- リファクタリング
- 質問・相談
- その他

ユーザー: 新機能

アシスタント: 優先度を選択してください:
- 最優先
- 高
- 中
- 低

ユーザー: 中

アシスタント: 問題の詳細を教えてください。

ユーザー: 管理画面のユーザーリストページから、ユーザー情報をCSV形式でエクスポートできる機能を追加したい。フィルタリングした結果もエクスポートできると便利。

アシスタント: 再現手順があれば教えてください（なければ「スキップ」と入力）。

ユーザー: スキップ

アシスタント: 期待する動作があれば教えてください（なければ「スキップ」と入力）。

ユーザー: 1. 管理画面のユーザーリストページに「CSVエクスポート」ボタンを追加
2. ボタンをクリックするとCSVファイルがダウンロードされる
3. フィルタリング後の結果もエクスポート可能

アシスタント: その他追加情報があれば教えてください（なければ「スキップ」と入力）。

ユーザー: 出力項目: ユーザーID, 名前, メールアドレス, 登録日, 最終ログイン日

アシスタント: 以下の内容でIssueを作成します:

---
# タイトル
ユーザーリストのCSVエクスポート機能を追加

## 問題の詳細
管理画面のユーザーリストページから、ユーザー情報をCSV形式でエクスポートできる機能を追加したい。フィルタリングした結果もエクスポートできると便利。

## 期待する動作
1. 管理画面のユーザーリストページに「CSVエクスポート」ボタンを追加
2. ボタンをクリックするとCSVファイルがダウンロードされる
3. フィルタリング後の結果もエクスポート可能

## 追加情報
出力項目: ユーザーID, 名前, メールアドレス, 登録日, 最終ログイン日

ラベル: Type：新機能・既存改修, Priority：中
---

この内容でIssueを作成しますか？ (yes/no)

ユーザー: yes

アシスタント: Issue #53 を作成しました: https://github.com/user/repo/issues/53
```

## 例5: 一度に複数情報を提供

### ユーザーの要望
ユーザーが最初から詳細を提供する場合

### 対話フロー

```
ユーザー: バグをIssueにしたい。タイトルは「検索結果が表示されない」で、検索ボックスにキーワードを入力しても何も表示されない問題です。優先度は高で。

アシスタント: 了解しました。以下の情報を確認させてください:
- タイトル: 検索結果が表示されない
- Issue種別: バグ
- 優先度: 高
- 詳細: 検索ボックスにキーワードを入力しても何も表示されない

再現手順があれば教えてください（なければ「スキップ」と入力）。

ユーザー: 1. トップページを開く
2. 検索ボックスに「テスト」と入力
3. 検索ボタンをクリック
4. 何も表示されない

アシスタント: 期待する動作があれば教えてください（なければ「スキップ」と入力）。

ユーザー: 「テスト」を含む記事一覧が表示される

アシスタント: その他追加情報があれば教えてください（なければ「スキップ」と入力）。

ユーザー: コンソールに404エラーが出ている

アシスタント: 以下の内容でIssueを作成します:

---
# タイトル
検索結果が表示されない

## 問題の詳細
検索ボックスにキーワードを入力しても何も表示されない

## 再現手順
1. トップページを開く
2. 検索ボックスに「テスト」と入力
3. 検索ボタンをクリック
4. 何も表示されない

## 期待する動作
「テスト」を含む記事一覧が表示される

## 追加情報
コンソールに404エラーが出ている

ラベル: Type：バグ, Priority：中
---

この内容でIssueを作成しますか？ (yes/no)

ユーザー: yes

アシスタント: Issue #56 を作成しました: https://github.com/user/repo/issues/56
```
