# Simple Issue Creator

ユーザーとの対話を通じて、シンプルにGitHub Issueを作成するスキルです。

## 概要

このスキルは、ユーザーから問題の内容を聞き取り、適切な形式でGitHub Issueを作成します。コード分析などの事前情報は不要で、ユーザーとの対話のみでIssueを作成できます。

## 主な機能

- **対話形式の情報収集**: タイトル、種別、優先度、詳細などを順番に質問
- **柔軟な入力**: 一度に複数の情報を提供することも可能
- **プレビュー機能**: Issue作成前に内容を確認
- **修正対応**: プレビュー後の修正にも対応
- **標準ラベル適用**: プロジェクトの標準ラベル体系を自動適用

## 使い方

### 基本的な使い方

1. スキルを起動:
   ```
   ユーザー: Issueを作成したい
   ```

2. 質問に答える:
   - タイトル
   - Issue種別（バグ/新機能/ドキュメント等）
   - 優先度
   - 問題の詳細
   - 再現手順（任意）
   - 期待する動作（任意）
   - 追加情報（任意）

3. プレビュー確認後、作成を承認

### トリガーフレーズ

以下のフレーズでスキルが起動します:

- "Issueを作成したい"
- "バグを報告したい"
- "機能追加のIssueを作りたい"
- "問題をIssueにして"

## Issue種別とラベル

| 種別 | 適用されるラベル |
|------|-----------------|
| バグ | Type：バグ |
| 新機能 | Type：新機能・既存改修 |
| ドキュメント | Type：ドキュメント更新 |
| リファクタリング | Type：リファクタリング |
| 質問・相談 | Type：質問・相談 |
| その他 | Type：作業 |

## 優先度とラベル

| 優先度 | 適用されるラベル |
|--------|-----------------|
| 最優先 | Priority：最優先 |
| 高 | Priority：中 |
| 中 | Priority：中 |
| 低 | Priority：低 |

**注意**: 質問・相談の場合は、優先度の代わりに緊急度ラベル（緊急度：高/中/低）が適用されます。

## 使用例

### 例1: シンプルなバグ報告

```
ユーザー: バグをIssueにしたい
スキル: Issueのタイトルを教えてください
ユーザー: ログイン機能が動作しない
スキル: Issue種別を選択してください...
[対話が続く]
スキル: Issue #52 を作成しました
```

### 例2: 詳細情報を一度に提供

```
ユーザー: バグをIssueにしたい。タイトルは「検索結果が表示されない」で、
         検索ボックスにキーワードを入力しても何も表示されない問題です。
         優先度は高で。
スキル: 了解しました。再現手順があれば教えてください...
[必要な情報のみ追加で質問]
```

詳細な使用例は [EXAMPLES.md](EXAMPLES.md) を参照してください。

## 前提条件

- GitHub CLIがインストールされている
- Gitリポジトリ内で実行
- リモートリポジトリが設定されている
- GitHub認証が完了している

## 作成されるIssueの構造

```markdown
## 問題の詳細
{ユーザーが入力した詳細}

## 再現手順
{ユーザーが入力した再現手順}

## 期待する動作
{ユーザーが入力した期待する動作}

## 追加情報
{ユーザーが入力した追加情報}
```

※ 任意項目がスキップされた場合、その項目は省略されます。

## 注意事項

- 任意項目は「スキップ」「なし」「-」等で省略可能
- プレビュー確認後に修正も可能
- Issue作成前に必ずプレビューで内容を確認
- 標準ラベルが未設定のプロジェクトでは、ラベル適用がスキップされる場合があります

## トラブルシューティング

### Issue作成に失敗する

- GitHub CLIがインストールされているか確認: `gh --version`
- GitHub認証が完了しているか確認: `gh auth status`
- リモートリポジトリが設定されているか確認: `git remote -v`

### ラベルが適用されない

- プロジェクトに標準ラベルが存在するか確認
- `/copy-labels`コマンドでラベルを適用してから再試行
