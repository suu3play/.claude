# Issue作成モード詳細

## モード1: コード分析Issue作成

### フロー
1. 分析結果のサマリー表示
2. Issue作成範囲の確認（全て/Critical/High/カテゴリ指定）
3. Issue粒度の確認（詳細/中/粗）
4. グループ化基準の確認（ファイル/機能/エラー種類/自動）
5. Issue内容の生成
6. ユーザー確認
7. GitHub Issue作成

### Issue内容生成ロジック

#### タイトル形式
```
[カテゴリ] 簡潔な説明 (影響範囲)
```

例:
- `[Bug] 欠落モデルファイルの作成 (Badge, Goal関連)`
- `[Refactoring] 未定義メソッドの実装 (AnswerRepository)`

#### 本文構成
`references/issue-templates.md` のコード分析Issue用テンプレートを使用

### グループ化基準

#### ファイル単位
同じファイルの問題をまとめる

#### 機能単位
同じ機能の問題をまとめる

#### エラー種類単位
同じエラータイプをまとめる

#### 自動判定（推奨）
関連性の高い問題を自動グループ化

## モード2: Good First Issue作成

### フロー
1. 初級者向けタスクの特定
2. タスクの評価と選定
3. Good First Issue内容の構成
4. ユーザー確認
5. GitHub Issue作成

### 初級者向けタスクのカテゴリ

#### 📝 ドキュメント改善
- READMEにインストール手順を追加
- API関数にJSDocコメントを追加
- コントリビューションガイドの作成

#### 🎨 UI/スタイル改善
- ボタンのホバー効果追加
- カラースキームの統一
- レスポンシブデザインの微調整

#### 🐛 簡単なバグ修正
- テキストのタイポ修正
- リンク切れの修正
- エラーメッセージの改善

#### 🧪 テスト追加
- 既存関数の単体テスト追加
- テストケースのカバレッジ向上

#### ♿ アクセシビリティ改善
- ボタンにaria-labelを追加
- 画像にalt属性を追加

#### 🔧 コードクリーンアップ
- 未使用のインポート削除
- console.logの削除
- フォーマットの統一

### 選定基準
- 難易度スコア ≤ 3 (1-5点)
- 影響範囲スコア ≤ 3 (1-5点)
- 学習価値スコア ≥ 2 (1-5点)

## モード3: 通常Issue作成

### フロー
1. Issue情報の対話的聞き取り
   - タイトル（必須）
   - Issue種別（任意、デフォルト: バグ）
   - 優先度/緊急度（任意、デフォルト: 中）
   - 問題の詳細（必須）
   - 再現手順（任意）
   - 期待する動作（任意）
   - 追加情報（任意）
2. Issue本文の構成
3. ユーザー確認
4. GitHub Issue作成

### Issue種別とラベルのマッピング
- バグ → `Type：バグ`
- 新機能 → `Type：新機能・既存改修`
- ドキュメント → `Type：ドキュメント更新`
- リファクタリング → `Type：リファクタリング`
- 質問・相談 → `Type：質問・相談`
- その他 → `Type：作業`

### 優先度/緊急度のマッピング

#### 通常のIssue
- 最優先 → `Priority：最優先`
- 高 → `Priority：中`
- 中 → `Priority：中`
- 低 → `Priority：低`

#### 質問・相談の場合
- 高 → `緊急度：高`
- 中 → `緊急度：中`
- 低 → `緊急度：低`

## 設定オプション

環境変数でカスタマイズ可能:

```bash
# デフォルトの粒度（fine/medium/coarse）
export ISSUE_DEFAULT_GRANULARITY=medium

# デフォルトの優先度フィルター
export ISSUE_DEFAULT_PRIORITY=critical,high

# 自動アサイン
export ISSUE_AUTO_ASSIGN=true

# マイルストーン
export ISSUE_DEFAULT_MILESTONE=v1.0

# レポート出力先
export ISSUE_REPORT_DIR=./reports

# レート制限の待機時間（秒）
export ISSUE_RATE_LIMIT_WAIT=1
```
